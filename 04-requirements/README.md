# Requirements

Each requirement lives in its own folder named `REQ-XXX-<slug>/`.

Inside every requirement folder:

- `requirement.md` - the requirement statement + acceptance criteria
- `meta.yml` - priority, status, owner, links to epic
- `user-stories/` - one file per user story (`US-XXX.md`)
- `test-cases/` - one file per test case (`TC-XXX.md`)
- `test-cases/test-results/` - test execution results (`TR-XXX.md`)

Cross-cutting:

- `epics/` - epic definitions + the story-to-epic mapping
